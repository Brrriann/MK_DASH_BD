## 0. 배경 (왜 새 세션인가)

- 노션은 **처음엔 정상 연결**됐고(도구 `mcp__Notion__*` 로드됨), 통합 허브 페이지까지 생성 완료.
- 이후 노션 커넥터만 끊긴 뒤로, 다른 커넥터(Canva·Gmail·Drive·GitHub·HIGGS)는 자동 복구되는데 **노션만 계속 빠짐**.
- 원인(가장 유력): **실행 중이던 세션이 "죽은 노션 연결"을 붙잡고 있어, 앱에서 재인증해도 그 세션엔 반영되지 않음.**
- 따라서 **현재 세션을 고치지 말고, 새 세션을 시작**하는 것이 가장 확실한 해결책.
- 참고: 혼선을 유발하던 중복 `.mcp.json`(`notion` 서버)은 제거 완료. 노션은 **앱 커넥터로 일원화**.

## 0.5 Git / 머지 상태 (중요 — 헷갈리지 말 것)

- **이 세션의 코드 변경 = 없음(net-zero).** `.mcp.json`을 추가했다가 제거 → `main`과 코드 diff 0. PR #15는 닫음.
- **앱 개발 작업(#8~#12: 비밀번호 재설정, 리드→고객 전환, 코드 전체 검수 Phase 1-3, 세금계산서 PDF 재설계 등)은 전부 이미 `main`에 머지됨.**
  - 작업 중 잠깐 "미머지"처럼 보인 건 **로컬 `origin/main` ref가 옛날(`4e84aeb`)** 이라 생긴 착시. `git fetch` 후 `origin/main = 08cd3c7` 확인, `08cd3c7..HEAD` **코드 diff 없음**.
- **노션 허브 작업은 노션에 저장**됨 → git 머지 대상이 아님 (세션이 바뀌어도 그대로 있음).
- **결론: 추가로 머지할 코드 작업 없음.** git에 보존할 대상은 **이 안내서 파일뿐** (원하면 PR로 `main`에 반영).
- 남은 앱 쪽 할 일은 코드가 아니라 **수동 작업**: `handoff.md`의 "다음 TODO" 참고 (예: `021_backfill_invoice_total_amount.sql` Supabase 실행, Storage 버킷 생성 등). 이건 노션 작업과 별개.
